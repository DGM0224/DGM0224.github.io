팀 성과

OneStack은 팀원들과의 본격적인 협업을 통해 처음으로 개발한 서비스로, 백엔드와 데이터 흐름에 대한 실무적인 감각을 익히는 소중한 경험이었습니다.
Java와 Spring Boot를 활용하여 실제 서비스를 구현하며 객체지향 설계, API 설계, 데이터베이스 연동 등 실무 개발에 대한 이해도를 크게 높일 수 있었습니다.
또한, Jira, Confluence 등을 통해 체계적인 협업을 처음 경험하면서, 기획부터 개발, 배포까지의 전체 프로세스를 팀 단위로 수행하는 역량을 기를 수 있었습니다.

개인 기여

본 프로젝트에서 백엔드 개발자로서 주요 기능 개발과 인프라 구축에 깊이 관여하였습니다.

  • Java + Spring Boot 기반으로 비동기 처리를 구현하여,
    사용자가 입력한 조건에 따라 Ajax 기반의 실시간 데이터 조회가 가능하도록 설계
    또한 IntersectionObserver API를 활용하여 무한 스크롤 기능을 구현함으로써
    데이터 과다 로딩을 방지하고 사용자 경험 향상
  • 전문가의 프로필 상세보기 페이지 및 견적 요청 기능을 설계 및 구현하였으며,
    RESTful API와 MyBatis를 통해 데이터 흐름을 연결하여 효과적으로 매칭
  • WebSocket을 활용한 실시간 채팅 기능 내에서 포트원 API를 연동하여
    신용카드 기반 결제 시스템을 구현하고, 이후 결제 내역 확인 페이지를 구성하여
    사용자 편의성과 신뢰성을 확보
  • MySQL을 기반으로 초기 데이터베이스 스키마 설계 및 관리를 주도하였으며,
    정규화로 데이터 중복을 최소화하고 ERD 작성 및 인덱스 최적화를 통해 성능 개선
  • AWS EC2를 활용하여 서비스 배포를 진행하고,
    Nginx를 활용해 정적 파일과 이미지에 대한 CDN 서버 구성을 담당
    또한 Docker + Jenkins 기반으로 CI/CD 파이프라인을 구축하여
    개발부터 배포까지의 프로세스를 자동화하고 팀의 개발 효율성을 향상
이 프로젝트를 통해 백엔드 개발자로서의 실무 능력을 체득하였고,
클라우드 환경에서의 배포 경험, 자동화된 DevOps 환경 구성, 데이터 흐름 최적화 등
여러 기술을 경험하며 향후 DevOps 및 데이터 엔지니어링 분야로 성장을 꿈꾸고 있습니다.

트러블 슈팅 및 해결 방안

비동기 무한스크롤 로딩 중 중복 데이터 표시

  • 스크롤을 빠르게 내리면 같은 데이터가 두 번씩 출력되는 문제 발생
    이를 해결하기 위해, 이미 조회된 데이터를 제외하고 조회하는 쿼리로 수정 및
    API 요청 중엔 추가 요청을 막는 로딩 스피너 사용

결제 금액 조작 방지 미비

  • 프론트에서 가격을 변경해 결제 요청을 보내는 악용 시도 발견
    이를 해결하기 위해, 백엔드에서 서버 기준으로 금액 검증하는 로직을 추가

결제 시스템 오류

  • 결제 시스템에서 발생한 결제 실패 문제는 외부 API와의 통신 오류로 인해 발생
    이를 해결하기 위해, 결제 처리 과정에서의 예외 처리를 강화하고,
    결제 실패 시 모든 과정을 취소하는 트랜잭션을 추가하여 안정성을 확보

정규화 부족으로 인한 데이터 중복

  • 전문가 정보가 여러 테이블에 중복 저장되어 관리 어려움 발생
    이를 해결하기 위해, 전문가 프로필 정보를 하나의 테이블로 분리하고,
    다른 테이블은 외래키 참조로 연결해 3정규형 기반으로 재설계

Nginx에서 정적 파일 미출력 (CDN 서버 문제)

  • 이미지가 로컬에선 잘 나오는데, 배포 서버에선 404 오류 발생
    Nginx에서 root 경로 설정이 잘못되어 있던 문제 확인
    이를 해결하기 위해, location 블록 수정 및 권한 문제 해결로 정상 동작

Jenkins 빌드 실패

  • GitHub에서 push하면 Jenkins 빌드가 자동 실행되지만, 종종 실패 발생
    빌드 시점에 사용 중인 포트가 열려 있어 컨테이너 충돌 발생 문제 확인
    이를 해결하기 위해, 기존 컨테이너를 중지하고 새로 빌드하도록 파이프라인 수정
